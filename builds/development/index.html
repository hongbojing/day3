HTML inline elements VS block elements

Half a year ago, I was asked ' what is the difference of "span" tag and "div" tag' , I answered that "span is for storing small things and div is for storing bigger things", because that's my sence for these two tags, after that interview, I googled them and found that span is a inline tag and div is a block tag, I know them, seriously, I know that span is an inline tag and div is a block element tag but for some reason I take that for granted, it was like you breathe the air everyday and you can not tell what are in the air, I think this is a common situation for those developers who just code everyday and do not prepare for the interviews (of course it was my fault that I did not prepare enough for that interview), so I create this page to write some examples about inline tags and block tags, for those who are preparing for an interview, you might use this as an example if you do not know what inline tags and block tags look like.

I googled an answer from HERE, which says: The difference between span and div is that a span element is in-line and usually used for a small chunk of HTML inside a line (such as inside a paragraph) whereas a div (division) element is block-line (which is basically equivalent to having a line-break before and after it) and used to group larger chunks of code.

Here is a list of block element tags, they all take a whole line space. Of course I use a block tag <ul><li></li></ul> to display them.

  • address,article, aside, audio
  • blockquote, canvas, dd, dl, fieldset, figcaption, figure, footer
  • h1,h2,h3,h4,h5,h6, header, hgroup, hr
  • main, nav, noscript, ol, output
  • p, pre, section, table, tfoot, ul, video

An inline element occupies only the space bounded by the tags that define the inline element. The following are inline element tags:

  • b, big, i, small, tt
  • abbr, acronym, cite, code, dfn, em, kbd, strong, samp, var
  • a, bdo, br, img, map, object, q, script, span, sub, sup
  • button, input, label, select, textarea
